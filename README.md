# Formulario

Aplicación React construida con Vite para crear y listar soluciones/tareas con validación de formulario y persistencia en Supabase.

## Funcionalidades

- Formulario con validación en tiempo real
- Validaciones de:
  - título entre 5 y 60 caracteres
  - descripción entre 20 y 500 caracteres
  - email válido
  - categoría obligatoria
  - prioridad entre 1 y 5
- Botón de envío deshabilitado si el formulario es inválido
- Estado visual de envío
- Prevención de doble envío
- Manejo de errores de red y permisos
- Reintento manual
- Persistencia de datos con Supabase
- Listado de soluciones filtrado por email
- Orden descendente por fecha de creación

## Tecnologías

## Instalación y ejecución

1. Clonar el repositorio:

```

```

2. Entrar en la carpeta del proyecto:

```

```

3. Instalar dependencias:

```

4. Crear archivo `.env` en la raíz con:

5. Ejecutar el proyecto:

```

## Base de datos

Tabla usada en Supabase:

```
create table if not exists public.soluciones (
  id bigint generated by default as identity primary key,
  titulo varchar(60) not null,
  descripcion varchar(500) not null,
  categoria varchar(100) not null,
  prioridad int not null check (prioridad >= 1 and prioridad <= 5),
  email varchar(255) not null,
  created_at timestamptz not null default now()
);
```

Políticas aplicadas:

```
alter table public.soluciones enable row level security;

create policy "Permitir insertar soluciones"
on public.soluciones
for insert
to anon
with check (true);

create policy "Permitir leer soluciones"
on public.soluciones
for select
to anon
using (true);
```

## Decisiones técnicas

- Se utilizó un hook personalizado para separar la lógica de validación del componente visual.
- Se usó Supabase como backend serverless para persistencia rápida y simple.
- El filtrado por “mis soluciones” se resolvió mediante email, ya que no se implementó autenticación real.
- El orden descendente se realiza desde la consulta a Supabase usando `created_at`.
